# 无损
1. 减小文件大小的同时，不丢弃任何原始数据信息，在解压后可以完美地、逐比特地恢复到原始状态。
2. 通过寻找数据中的统计冗余（例如，重复的颜色或模式）并用更紧凑的方式来表示。
## BMP (Windows Bitmap)
### 概述
1. 微软 Windows 操作系统中使用的标准图像文件格式。
2. 通常不进行压缩，也可以使用简单的RLE无损压缩。
### 优点
1. 易于编程和处理，与 Windows 平台高度兼容。
### 缺点
1. 由于几乎不压缩，BMP 文件是所有格式中最大的，完全不适用于网络。
2. 不支持透明度、图层等现代图像特性。
### 适用场景
主要用于Windows系统下的图标和壁纸，以及一些对图像质量有极端要求但不在乎存储空间的特定应用。在现代应用中已较少使用。
## GIF (Graphics Interchange Format)
### 概述
LZW 算法，至今仍因其独特的动画功能而活跃在互联网上。
### 优点
1. 能够在一个文件中存储多个帧，从而创建简单的循环动画，广泛用于网络表情包和广告。
2. 由于颜色数量有限，对于色彩简单的图像，文件体积非常小。
3. 支持索引透明，即可以将调色板中的一种颜色指定为透明。
### 缺点
1. 最多只支持 8 位颜色，表现色彩丰富的照片会有严重的颜色失真。
2. 只有全透明或不透明两种状态，不支持半透明。
3. 对于现代图像，其压缩算法效率已不如PNG和WebP。
### 适用场景
简单的网络动画（表情包）、颜色数量少的Logo和图标。
## PNG (Portable Network Graphics)
### 概述
1. 被设计用来替代 GIF，并提供了更多的功能，尤其在网页设计领域备受青睐。
2. Deflate算法（与ZIP压缩相同），在压缩和解压过程中，图像的任何细节都不会丢失。
### 优点
1. 无论保存多少次，图像质量都保持不变，适合需要编辑的图像。
2. 支持完整的 Alpha 通道，可以实现 256 个级别的半透明效果，使得图像可以完美地融入任何背景。
3. 支持24位真彩色，色彩还原准确。
### 缺点
1. 相较于 JPEG，对于色彩复杂的照片类图像，PNG 文件通常会大很多。
2. 标准的 PNG 格式不支持动画（其扩展格式 APNG 支持，但兼容性不如 GIF）。
3. 通常不支持用于专业印刷的 CMYK 颜色模式。
### 适用场景
需要透明背景的Logo、图标、网页上的图表和插图、需要保证质量的截图。
# 有损
通过永久性地丢弃一部分数据来获得远高于无损压缩的压缩比，选择性地移除那些对人类视觉系统来说最不敏感的信息。
## JPEG (Joint Photographic Experts Group)
### 概述
1. 通常写作 .jpg，是目前最流行的图像格式，尤其在照片存储和网络分享方面。
2. 基于人眼对色彩的敏感度低于对亮度的敏感度特性，移除一部分图像数据以减小文件大小。
### 优点
1.  能够在保持可接受图像质量的同时，极大地减小文件体积，非常适合网络传输和存储。
2.  支持 24 位真彩色，非常适合表现色彩层次丰富的照片。
3.  兼容性好，几乎所有的设备和软件都支持 JPEG 格式。
### 缺点
1. 每次保存都会导致图像质量下降，不适合需要反复编辑的图像。
2. JPEG 无法处理透明或半透明的背景。
3. 对于包含清晰线条、文字或大面积纯色的图像（如Logo、图表），压缩后容易在边缘产生模糊或“振铃”效应的噪点。
### 适用场景
数码照片、写实风格的图片、需要控制文件大小的网页图片。
# 可选
## TIFF (Tagged Image File Format)
### 概述
1. 一种非常灵活的图像格式，在专业摄影、桌面出版和图形设计领域是行业标准。
2. “容器”格式，可以使用不同压缩算法（包括无损的 LZW、ZIP 或有损的 JPEG）。其核心是保留图像的最高质量。
### 优点
1. 主要采用无损压缩，能够完美保留所有原始图像信息。
2. 支持图层、Alpha 通道、路径等多种复杂数据，方便后期编辑。
3. 支持RGB、CMYK等多种颜色模式，是专业印刷的首选格式。
4. 一个TIFF文件可以包含多个页面，常用于扫描文档的归档。
### 缺点
1. 由于保留了大量数据，TIFF 文件体积非常庞大，不适合在网络上传输。
2. 虽然是标准格式，但并非所有浏览器和简单的图像查看器都能完全支持其所有高级功能。
### 适用场景
专业摄影后期处理、高质量的印刷品、医疗影像、文档扫描和存档。
## WebP
### 概述
1.  Google 开发的一种现代图像格式，旨在加速网页加载速度，它同时提供了优秀的有损和无损压缩能力。
2.  其有损压缩基于 VP8 视频编解码器，无损压缩则采用了新的算法。
### 优点
1. 在相同的图像质量下，WebP 的有损文件比 JPEG 小 25-34%，无损文件比 PNG 小约 26%。
2.  同时支持有损和无损压缩、Alpha 透明通道以及动画。
### 缺点
1.  尽管所有现代主流浏览器（Chrome, Firefox, Edge, Safari）均已支持，但一些老旧的浏览器版本或某些图像编辑软件可能不兼容。
2.  在极高压缩比下，有损WebP可能会比JPEG产生更多的细节平滑或模糊。
### 适用场景
几乎所有网页图像，包括照片、图标、插图和动画，是未来网页图像格式的主流趋势。
# 矢量图
## SVG (Scalable Vector Graphics)
1. 基于XML的开放标准，是网页矢量图形的首选格式。
2. 可以被浏览器直接渲染，并能通过CSS和JavaScript进行交互操作。
## AI (Adobe Illustrator)
Adobe Illustrator软件的原生文件格式，是矢量图形创作和编辑过程中的标准格式。
## EPS (Encapsulated PostScript)
较早的跨平台矢量图形交换格式，在印刷行业中有悠久历史，但正逐渐被 PDF 取代。
## PDF (Portable Document Format)
极其通用的文件格式，既可以包含矢量图形和文本，也可以嵌入光栅图像，是数字文档分发和印刷预备的标准。
